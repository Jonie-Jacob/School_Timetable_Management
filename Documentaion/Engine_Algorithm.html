2 Teacher Time Partitioning

whole_school_loader.py → compute_teacher_partitions()

T
Purpose

Prevent shared teachers from being consumed entirely by early-processed divisions. Each division gets a guaranteed pool of time slots for its shared teachers.

◆ For each teacher

Single-division teacher?

→ YES: Give ALL available slots (no partition needed)

Shared across 2+ divisions?

→ Collect all available time slots (excluding unavailability)

→ Check HARD preferences per division

→ Division HAS HARD prefs: Partition = only HARD-preferred slots

→ Division has NO HARD prefs: Partition = ALL available slots (non-exclusive)

Ex
Example: Anu Mathew (22pw across 8 divisions)

XI-BNoneAll 40 available slots

3 Demand-Driven Placement

greedy.py → schedule_all() main loop

!
Core Principle

Instead of sorting assignments then placing sequentially, always resolve the tightest bottleneck first and assign it the least constraining value.

↻
Main Loop
For EACH unplaced assignment-period, count valid slots
Pick the one with FEWEST valid slots (most constrained)
Score each valid slot by demand (how many others want it)
Place in slot with LOWEST demand (least constraining)
Update teacher_busy, chromosome, history
If 0 valid slots → Step 4 (Backtrack)

Valid Slot Checks

A slot at gene index gi is valid for assignment la if ALL pass:

#CheckSource
1Slot is empty (chromosome[gi] == -1)Chromosome state
2Day not in excludedDaysHARD preferences
3Day in preferredDays (if set)HARD preferences
4Period not in excludedPeriodRangeHARD preferences
5Period in preferredPeriodRange (if set)HARD preferences
6Teacher not unavailable at (day, time)Teacher availability
7Teacher not busy at (day, time)Global teacher_busy set
8Slot in teacher's partition for this divisionTeacher partitioning

Slot Scoring (Demand)

#
_compute_slot_demand()
demand = 0 # Count how many OTHER assignments in this division could use this slot for each other unplaced assignment: if teacher not busy AND not unavailable at this time: demand += 1 # Adjustments: if adjacent to same subject: demand -= 10 # strong adjacency preference if same day already has this subject: demand += 2 # spread penalty # Pick slot with LOWEST demand (save contested slots for those who need them)

4 Backtracking

greedy.py → _try_backtrack() — triggered when Step 3 finds 0 valid slots

↩
Undo & Retry (max depth 5)
Assignment X has 0 valid slots — teacher busy everywhere
Find last 50 placements involving same teacher(s)
Pick up to 5 candidates to try undoing
◆ For each undo candidate

a. UNDO the candidate (remove from chromosome, free teacher slot)

b. Try placing X again (teacher now free)

→ X fits? Place X, then re-place undone candidate elsewhere

  → Candidate also fits? SUCCESS

  → Candidate doesn't fit? Undo both, try next candidate

→ X still doesn't fit? Restore candidate, try next

All exhausted? → FALLBACK Place X in any empty slot (accepts conflict)

5 Local Optimization

greedy.py → _local_optimize() — deterministic swaps, no randomness

⇔
For each division → each day → each pair (i, j)
Would swapping create a teacher conflict? YES → Skip
Compute soft score before and after swap
Score improved? → Keep swap
Score same/worse? → Revert swap
